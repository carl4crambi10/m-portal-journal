# m-portal-journal

## 项目简介

**m-portal-journal** 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态 HTML 页面的集合与展示空间。

每个页面可视为独立的“日志条目”或“入口页面”，适合存放各类简单的网页内容、信息卡片或轻量级应用界面。

## 目录结构

```
m-portal-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 公共资源（CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- **pages/**：每个 `.html` 文件代表一个独立页面，命名建议清晰可辨。
- **assets/**：存放页面间共享的样式、脚本或媒体文件，避免重复引用。
- **index.html**：可选的入口页面，用于导航或展示所有页面的链接。

## 页面归档说明

- 所有页面均为纯静态 HTML，不含后端依赖。
- 页面之间相互独立，互不干扰，便于单独访问或引用。
- 归档内容可以是文档、工具界面、信息展示页等任意形式的 HTML。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/m-portal-journal.git
   ```
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 通过浏览器直接打开 `pages/` 中的文件，或通过本地服务器预览。
4. 提交更改并推送至远程仓库，即可通过 GitHub Pages 或其他静态托管服务发布。
